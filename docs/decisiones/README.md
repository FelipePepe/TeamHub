# Módulos de `decisiones.md`

Este directorio modulariza el contenido histórico y operativo de decisiones del proyecto.

## Estructura
- `00-contexto-e-indice-original.md`: Introducción e índice original (snapshot).
- `01-adrs-por-categoria.md`: ADRs por categorías funcionales (documentación, arquitectura, seguridad, frontend, backend, testing y devops).
- `02-adrs-registro-ejecucion.md`: ADRs de ejecución/operación (GitFlow, CI/CD, husky, colaboración LLM).
- `03-registro-fases-y-tareas.md`: Registro de ejecución por fases e historial detallado de tareas.
- `04-progreso-releases-cobertura.md`: Progreso global, releases, cobertura y ADRs recientes.
- `decisiones_legacy_full.md`: Backup completo previo a la modularización.

## Regla de mantenimiento
- `docs/decisiones.md` se mantiene como **índice maestro** para navegación rápida.
- El detalle se actualiza en el módulo correspondiente.
- Si se añade un ADR nuevo, actualizar también el índice maestro.
